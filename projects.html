Project SteinLine

Overview: Project SteinLine is an asymmetrical distributed forensic platform for processing large volumes of documents, images and video evidence using multimodal AI. It combines local/edge storage optimised for Raspberry Pi (SSD) with high-density GPU compute for inference and an interactive Qt-based visualization frontend.

Core architecture:

  • Storage & Registry — optimised for local/edge (Raspberry Pi 5 with SSD) setups.
  • Compute & Inference — designed for high-memory NVIDIA GPUs (RTX 3090/4090) using vLLM kernels.
  • Visualization — an interactive 2D infinite canvas built with Qt (PySide6).

Features: recursive windowing for long documents, a multimodal pipeline (PDF deconstruction, OCR, audio transcription), and a non-destructive workflow where source evidence remains read-only.

Prerequisites: Linux (Ubuntu recommended), Python 3.11+, and an NVIDIA GPU for full performance (24GB+ VRAM recommended).

Repository

Good for: practitioners building forensic pipelines, multimodal research prototypes, and anyone needing scalable evidence-processing workflows.

BadAppleStein

Overview: BadAppleStein is a Python/C project that reconstructs high-contrast video animations by matching frames to a library of PDF pages. It uses FFmpeg for video processing and provides scripts for arranging and rendering frames into final outputs.

Highlights:

  • Efficient matching routines (C components and a shared library) for frame-to-PDF lookup.
  • Multiple rendering pipelines (greedy, hyper, turbo) exposed as Python scripts.
  • Includes a demo video linked from the repository.

Good for: multimedia hackers, creative coding projects, and experimenting with content-aware rendering pipelines.
